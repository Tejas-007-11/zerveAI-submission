# SuccessLens: Understanding Behaviors That Drive Long-Term User Success

## 1. Introduction

Modern data platforms generate large volumes of user interaction events. Understanding how users interact with such platforms is critical for identifying what drives long-term engagement and success.

This project analyzes event-level behavioral data from Zerve to identify patterns of user activity that correlate with successful product usage. By combining exploratory analysis, feature engineering, and machine learning models, the project aims to uncover the key behavioral drivers of successful users.

The ultimate goal is to transform raw event logs into actionable insights that help improve user onboarding, engagement, and retention.

---

## 2. Problem Statement

Zerve provided a dataset containing application event data describing user activities across notebooks, runs, deployments, and AI agent interactions.

The core question addressed in this project is:

**Which user behaviors and workflows are most predictive of long-term success?**

To answer this question, we analyze user interactions, construct behavioral features, and build predictive models to identify the strongest drivers of success.

---

## 3. Dataset Description

The dataset consists of event-level tracking data generated by users interacting with the Zerve platform.

Each row in the dataset represents a user action captured as an event.

### Key Fields

| Field | Description |
|------|-------------|
| distinct_id | Unique identifier for each user |
| event | The action performed by the user |
| timestamp | Time when the event occurred |
| prop_$session_id | Session identifier |
| prop_tool_name | Tool or feature used |
| prop_credit_amount | Credits associated with actions |
| prop_credits_used | Credits consumed by the user |

The dataset also includes metadata such as browser information, device type, and geographic data. These fields were not central to behavioral analysis and were excluded during preprocessing.

---

## 4. Success Definition

To analyze successful usage patterns, we define **user success** as:

> A user who consumes platform credits during their interaction with the system.

Credit usage indicates deeper engagement with the platform's capabilities, suggesting meaningful usage beyond basic exploration.

A binary label is therefore created:

- **Success = 1** → User consumed credits
- **Success = 0** → User did not consume credits

---

## 5. Methodology

The analysis pipeline consists of the following stages:

### 1. Data Cleaning

- Loaded the dataset into a Python environment
- Converted timestamp fields into datetime format
- Removed irrelevant metadata columns
- Sorted events chronologically

### 2. Feature Engineering

Raw event logs were aggregated at the user level to create behavioral features.

Key features include:

- Total number of events
- Number of unique event types
- Number of sessions
- Active days
- Credits consumed
- Frequency of specific platform actions

Event counts were also transformed into feature vectors using event frequency tables.

### 3. Exploratory Data Analysis

Exploratory analysis was conducted to understand:

- Event distribution
- Most common user interactions
- Session patterns
- Activity levels across users

Visualizations were used to identify early patterns in user behavior.

### 4. Machine Learning Modeling

A supervised learning approach was used to predict successful users.

Models tested:

- Random Forest Classifier
- XGBoost Classifier

Evaluation metrics included:

- Precision
- Recall
- F1-score
- ROC-AUC

### 5. Feature Importance Analysis

Tree-based models allow extraction of feature importance scores, which help identify which behaviors most strongly influence user success.

---

## 6. Key Behavioral Insights

The analysis reveals several important behavioral signals associated with successful users:

### High Activity Levels

Users who generate more events and interact with multiple features show significantly higher success rates.

### Consistent Engagement

Users with multiple sessions and activity across several days demonstrate stronger long-term engagement.

### AI Agent Interaction

Interactions involving AI agent features (such as accepting suggestions or creating agent workers) are correlated with deeper platform usage.

### Tool Usage Diversity

Users who explore multiple tools within the platform are more likely to transition from exploration to productive usage.

---

## 7. Success Prediction Model

A machine learning model was developed to predict the probability of a user becoming successful.

The model uses behavioral features extracted from event logs.

The best performing model achieved strong predictive performance using XGBoost, indicating that behavioral signals captured in the dataset are highly informative.

The model produces a **Success Probability Score**, which can be used to segment users into:

- High probability of success
- Medium probability of success
- At-risk users

---

## 8. Product Insights and Recommendations

The results suggest several opportunities for improving user success on the platform.

### Improve Early Engagement

Encourage users to interact with key platform features early in their onboarding journey.

### Promote AI Agent Usage

AI-assisted features appear to correlate with deeper engagement, suggesting they should be highlighted during onboarding.

### Encourage Multi-Session Usage

Retention strategies should focus on bringing users back for additional sessions.

### Feature Discovery

Users who explore multiple tools are more likely to succeed, suggesting that guided feature discovery may increase engagement.

---

## 9. Deployment

The trained machine learning model can be deployed within the Zerve platform to provide real-time success predictions.

Potential applications include:

- Identifying users at risk of disengagement
- Triggering targeted onboarding guidance
- Personalizing product recommendations

This enables a proactive approach to improving user success.

---

## 10. Conclusion

This project demonstrates how event-level behavioral data can be transformed into actionable product insights.

By combining exploratory analysis, feature engineering, and predictive modeling, we identified key behavioral signals that indicate successful user engagement.

The SuccessLens framework provides a scalable approach to understanding user behavior and can help guide product decisions aimed at improving long-term user success.
